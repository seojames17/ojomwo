## 오점뭐(App) 설계 문서: 오늘 점심 뭐먹지

### 목표
- 선택장애가 있는 사람이 `현재 위치 주변`에서 `가까운 곳 우선`으로 후보를 몇 개만 뽑고,
- 그중에서 랜덤으로 최종 추천을 받아 쉽게 결정을 돕는 앱

### 사용자 흐름 (MVP)
1. 앱 진입
2. 위치 입력(현재 대략 위치 - IP)
3. 조건 설정(반경, 후보 풀 크기, 가까운 후보 우선 범위, 뽑을 개수)
4. 버튼 클릭: `오늘 점심 랜덤 추천`
5. 후보 몇 곳을 카드 형태로 보여주고, 지도 링크를 제공
6. `다시 뽑기`로 같은 조건에서 다른 조합을 빠르게 확인

### MVP 핵심 로직
- 주변 장소 후보를 먼저 `N개 이상` 가져온 뒤,
- 후보에 `distance(거리)` 정보가 있으면 가까운 순으로 정렬
- 상위 `top_percent` 범위를 후보 풀로 제한하고,
- (설정에 따라) 가까울수록 더 뽑힐 확률로 랜덤 추출해 `pick_count`개를 추천
- 룰렛(아케이드) 애니메이션으로 후보 스핀 후 결과 표시

### 데이터 소스(장소 후보 검색) 전략
기존 프로젝트에 이미 있는 장소/지오코딩 로직을 재사용합니다.
- `dining_app/place_provider.py`의 `search_nearby_places()`를 호출
- 후보 소스는 기본 `kakao`를 사용하고, Kakao 키가 없으면 `stub`으로 동작합니다.

### 필요한 환경 변수(키를 실제로 쓰고 싶을 때만)
- `KAKAO_REST_API_KEY` (카카오 로컬)

### 구현 파일 구성(권장)
- `ojomwo_app.py`: 실제 Streamlit UI 및 추천 로직
- `dining_app/place_provider.py`: 주변 장소 검색(카카오 + stub)

### 실행 방법
- 개발 서버 실행:
  - `streamlit run ojomwo_app.py`
- 키 설정(선택): `.streamlit/secrets.toml.example`을 참고해 같은 폴더에 `secrets.toml` 생성, 또는 환경변수 `KAKAO_REST_API_KEY` 설정
- 주의:
- API 키가 없으면 `stub` 후보가 사용되어 “가까운 곳 우선” 의미가 데모 수준으로 제한됩니다.
- 위치(IP) 및 장소검색은 인터넷 연결이 필요합니다.
